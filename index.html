Estructura básica del documento

<html>Tradicional

Elemento raíz de una página HTML. Dentro de él se coloca todo el contenido del documento.

<html lang="es">
  ...
</html>
<head>Tradicional

Contiene información que no se muestra directamente: título, estilos, metadatos y enlaces externos.

<head>
  <title>Mi web</title>
</head>
<body>Tradicional

Contiene todo lo visible de la página: textos, imágenes, botones, enlaces y secciones.

<body>
  <h1>Hola HTML</h1>
</body>
<title>Tradicional

Define el título que aparece en la pestaña del navegador y ayuda al posicionamiento SEO.

<title>Guía de HTML</title>
<meta>Tradicional

Aporta metadatos como la codificación, la descripción o la adaptación a móviles.

<meta charset="UTF-8">
<meta name="viewport" content="width=device-width, initial-scale=1.0">
<link>Tradicional

Conecta el HTML con recursos externos, como hojas de estilo CSS o iconos.

<link rel="stylesheet" href="estilos.css">

Etiquetas semánticas

<header>Semántica

Representa la cabecera de una página o sección. Suele incluir logo, título o navegación.

<header>
  <h1>Mi sitio web</h1>
</header>
<nav>Semántica

Agrupa enlaces de navegación importantes, como el menú principal de una web.

<nav>
  <a href="#inicio">Inicio</a>
</nav>
<main>Semántica

Indica el contenido principal y único de la página. Normalmente solo debe haber un main.

<main>
  <h2>Contenido principal</h2>
</main>
<section>Semántica

Define una sección temática dentro del documento, normalmente con su propio encabezado.

<section>
  <h2>Servicios</h2>
</section>
<article>Semántica

Representa contenido independiente, como una noticia, entrada de blog o tarjeta de producto.

<article>
  <h2>Noticia</h2>
  <p>Texto de la noticia.</p>
</article>
<aside>Semántica

Contiene información complementaria, como una barra lateral, anuncios o enlaces relacionados.

<aside>
  <p>Contenido relacionado</p>
</aside>
<footer>Semántica

Define el pie de página o de una sección. Suele incluir autoría, contacto o enlaces legales.

<footer>
  <p>© 2026 Clase HTML</p>
</footer>
<figure>Semántica

Agrupa contenido ilustrativo, como una imagen, gráfico o fragmento de código.

<figure>
  <img src="foto.jpg" alt="Paisaje">
</figure>
<figcaption>Semántica

Añade una explicación o pie de imagen dentro de una etiqueta figure.

<figcaption>Imagen del proyecto</figcaption>

Texto, listas y enlaces

<h1>...<h6>Tradicional

Definen títulos y subtítulos. h1 es el más importante y h6 el menos importante.

<h1>Título principal</h1>
<h2>Subtítulo</h2>
<p>Tradicional

Crea un párrafo de texto. Es una de las etiquetas más usadas en HTML.

<p>Este es un párrafo.</p>
<a>Tradicional

Crea enlaces hacia otras páginas, secciones, archivos o direcciones externas.

<a href="https://developer.mozilla.org">Aprender HTML</a>
<strong>Tradicional

Marca texto con importancia especial. Visualmente suele mostrarse en negrita.

<strong>Importante:</strong> guarda tu archivo.
<em>Tradicional

Indica énfasis en una palabra o frase. Visualmente suele mostrarse en cursiva.

<em>Esto debe leerse con énfasis.</em>
<span>Tradicional

Contenedor en línea para aplicar estilos o seleccionar una parte concreta del texto.

<p>Precio: <span>19,90 €</span></p>
<div>Tradicional

Contenedor genérico de bloque. Se usa cuando no existe una etiqueta semántica más adecuada.

<div class="tarjeta">
  <p>Contenido</p>
</div>
<ul>Tradicional

Crea una lista sin orden numérico, normalmente con viñetas.

<ul>
  <li>HTML</li>
  <li>CSS</li>
</ul>
<ol>Tradicional

Crea una lista ordenada, normalmente numerada.

<ol>
  <li>Abrir VS Code</li>
  <li>Crear index.html</li>
</ol>
<li>Tradicional

Representa cada elemento dentro de una lista ul u ol.

<li>Elemento de lista</li>
<br>Tradicional

Inserta un salto de línea. Conviene usarlo solo cuando sea realmente necesario.

Primera línea<br>
Segunda línea
<hr>Tradicional

Inserta una separación temática entre bloques de contenido.

<p>Tema 1</p>
<hr>
<p>Tema 2</p>

Imágenes, multimedia y tablas

<img>Tradicional

Inserta una imagen. El atributo alt describe la imagen y mejora la accesibilidad.

<img src="foto.jpg" alt="Alumno programando">
<picture>Semántica

Permite cargar diferentes imágenes según el tamaño de pantalla o el formato compatible.

<picture>
  <source srcset="foto.webp" type="image/webp">
  <img src="foto.jpg" alt="Ejemplo">
</picture>
<video>Tradicional

Inserta un vídeo en la página. Puede incluir controles de reproducción.

<video controls width="400">
  <source src="video.mp4" type="video/mp4">
</video>
<audio>Tradicional

Inserta audio, como música, narraciones o podcasts.

<audio controls>
  <source src="audio.mp3" type="audio/mpeg">
</audio>
<iframe>Tradicional

Incrusta contenido externo, como un vídeo de YouTube, un mapa o una página web.

<iframe src="https://www.example.com" title="Ejemplo"></iframe>
<table>Tradicional

Crea una tabla para mostrar datos organizados en filas y columnas.

<table>
  <tr><td>Dato</td></tr>
</table>
<tr>Tradicional

Representa una fila dentro de una tabla.

<tr>
  <td>Producto</td>
  <td>Precio</td>
</tr>
<th>Tradicional

Define una celda de encabezado en una tabla.

<th>Nombre</th>
<td>Tradicional

Define una celda normal de datos dentro de una tabla.

<td>María</td>

Formularios

<form>Tradicional

Agrupa campos para recoger y enviar datos del usuario.

<form action="/enviar" method="post">
  ...
</form>
<label>Tradicional

Asocia un texto descriptivo a un campo de formulario. Mejora la accesibilidad.

<label for="nombre">Nombre</label>
<input>Tradicional

Crea campos de entrada: texto, correo, contraseña, número, fecha, checkbox, radio, etc.

<input type="email" id="email" placeholder="tu@email.com">
<textarea>Tradicional

Crea un campo para escribir textos largos, como mensajes o comentarios.

<textarea rows="4" placeholder="Escribe tu mensaje"></textarea>
<select>Tradicional

Crea una lista desplegable de opciones.

<select>
  <option>HTML</option>
  <option>CSS</option>
</select>
<option>Tradicional

Define una opción dentro de un elemento select.

<option value="html">HTML</option>
<button>Tradicional

Crea un botón para enviar formularios o ejecutar acciones con JavaScript.

<button type="submit">Enviar</button>
Consejos para clase

Buenas prácticas al escribir HTML

HTML no solo consiste en que “se vea algo”. También debe estar ordenado, ser accesible, comprensible y fácil de mantener.

1. Usa etiquetas semánticas

Cuando el contenido sea una cabecera, un menú, una sección o un artículo, usa header, nav, section o article antes que abusar de div.

2. Respeta la jerarquía de títulos

Utiliza h1 para el título principal y h2, h3, h4 para organizar el contenido de forma lógica.

3. Añade textos alternativos

Las imágenes importantes deben tener atributo alt para que la página sea más accesible.

4. Separa estructura y diseño

HTML organiza el contenido. CSS se encarga del aspecto visual. JavaScript añade interacción.

5. Escribe código ordenado

Indenta bien las etiquetas. Un código limpio se entiende mejor y se corrige más rápido.

6. Prueba siempre en móvil

La mayoría de usuarios navegan desde pantallas pequeñas, así que revisa que todo se lea correctamente.
